# 📦 СкладПро — Система складского учёта

Полноценная система складского учёта с кассой, сканером штрихкодов и термопринтером.

## ✨ Возможности

- 📊 **Dashboard** — KPI, остатки, уведомления о низком запасе
- 📦 **Товары** — каталог с категориями, SKU, штрихкодами
- 📥 **Приход** — накладные от поставщиков с подтверждением
- 📤 **Расход** — отгрузка товаров с подтверждением
- 📋 **Остатки** — складские остатки в реальном времени
- 🛒 **Касса / POS** — продажа, оплата наличными/картой
- 🖨️ **Термопринтер** — печать чеков (ESC/POS / Web Print)
- 🔫 **Сканер штрихкодов** — поддержка USB/BT HID сканеров
- 📊 **Отчёты** — экспорт в JSON и Excel
- 🏭 **Склады** — мульти-складской учёт
- 👥 **Пользователи** — роли: admin / manager / operator
- 💾 **Система** — мониторинг диска, очистка данных

## 🚀 Деплой на Railway

### 1. Создать репозиторий на GitHub
```bash
git init
git add .
git commit -m "initial commit"
gh repo create warehouse-pro --public --push
```

### 2. Деплой на Railway
1. Зайти на [railway.app](https://railway.app)
2. New Project → Deploy from GitHub repo → выбрать репозиторий
3. Добавить сервис **PostgreSQL** (Add Service → Database → PostgreSQL)
4. Railway автоматически установит `DATABASE_URL`
5. В настройках сервиса добавить переменную:
   ```
   JWT_SECRET=your_random_secret_here
   ```
6. Готово! Railway запустит билд и деплой автоматически.

### 3. Первый вход
- URL: `https://your-app.railway.app`
- Логин: `Admin787`
- Пароль: `admin1221`

> ⚠️ **После первого входа смените пароль!**

## 💻 Локальная разработка

### Требования
- Node.js 18+
- PostgreSQL (или Docker)

### Установка
```bash
# Клонировать репозиторий
git clone https://github.com/your/warehouse-pro.git
cd warehouse-pro

# Установить зависимости
npm run install:all

# Настроить переменные окружения
cp backend/.env.example backend/.env
# Отредактировать backend/.env — указать DATABASE_URL

# Создать таблицы и заполнить начальными данными
npm run db:push
npm run db:seed

# Собрать фронтенд
npm run build

# Запустить
npm start
```

Или в режиме разработки (два терминала):
```bash
npm run dev:backend   # терминал 1
npm run dev:frontend  # терминал 2
```

## 🔫 Сканер штрихкодов

Сканеры работают как HID-клавиатура — без драйверов. Поддерживаются:
- Honeywell, Zebra, Symbol, Datalogic
- Любой USB/Bluetooth HID сканер

Просто наведите сканер в любое поле SKU или на экране POS — символы введутся автоматически + Enter.

## 🖨️ Термопринтер

**Метод 1 — Web USB (прямое подключение):**
- Chrome/Edge → при нажатии "Напечатать чек" появится диалог выбора USB устройства
- Выбрать принтер (Epson, АТОЛ, Bixolon, etc.)
- Работает с любым ESC/POS принтером

**Метод 2 — Браузерная печать (резерв):**
- Откроется окно браузера с чеком формата 80мм
- Выбрать принтер в диалоге печати
- Установить бумагу 80мм, убрать поля

## 📊 Роли пользователей

| Роль | Возможности |
|------|-------------|
| `admin` | Всё включая настройки системы, пользователей, складов |
| `manager` | Товары, приходы, расходы, отчёты, экспорт |
| `operator` | Приходы, расходы, касса, остатки (своего склада) |

## 🗄️ База данных

- `users`, `warehouses`, `products`, `categories`
- `suppliers`, `stock`, `receipts`, `receipt_items`
- `issues`, `issue_items`, `audit_log`

## 💡 Мониторинг диска

- **70%** — предупреждение в шапке
- **90%** — критическое уведомление
- **95%** — блокировка записи (кроме экспорта и очистки)

При заполнении: Система → скачать JSON → очистить старые данные.
