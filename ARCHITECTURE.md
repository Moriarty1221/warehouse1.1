## Описание ролей

### 👑 Администратор (`admin`)
Полный доступ ко всей системе без ограничений.

### 📋 Менеджер (`manager`)
Работа со складом, товарами, поставщиками, приходами, расходами, перемещениями, инвентаризацией, кассой, сменами, возвратами и отчётами. Не имеет доступа к настройкам системы, управлению складами и пользователями.

### 💳 Кассир (`cashier`)
Работает только с кассой. Может открыть/закрыть смену, проводить продажи и оформлять возвраты. Касса доступна только при открытой смене.

### 🏦 Инкассатор (`collector`)
Видит смены, историю продаж и отчёты (Z-отчёт, остатки). Не может проводить продажи, изменять товары или склад.

### 📦 Инвентор (`inventor`)
Проводит инвентаризацию. Может просматривать товары и остатки на складе. Не имеет доступа к кассе, продажам и управлению.

## Как ссылаться на разделы при изменениях

При обращении к ИИ просто укажите нужный раздел:

- **«Раздел 1»** — изменить логику JWT/авторизации → `backend/src/middleware/auth.js`
- **«Раздел 2»** — добавить поле к пользователям → `backend/src/routes/users.js`
- **«Раздел 3»** — изменить логику смен → `backend/src/routes/shifts.js`
- **«Раздел 4»** — изменить приходы → `backend/src/routes/receipts.js`
- **«Раздел 5»** — изменить расходы → `backend/src/routes/issues.js`
- **«Раздел 6»** — изменить товары → `backend/src/routes/products.js`
- **«Раздел 7»** — изменить остатки → `backend/src/routes/stock.js`
- **«Раздел 8»** — изменить отчёты → `backend/src/routes/reports.js`
- **«Раздел 9»** — изменить инвентаризацию → `backend/src/routes/inventory.js`
- **«Раздел 10»** — изменить перемещения → `backend/src/routes/transfers.js`
- **«Раздел 11»** — изменить возвраты → `backend/src/routes/returns.js`
- **«Раздел 12»** — изменить кассу/POS → `backend/src/routes/pos.js`
- **«Раздел 13»** — изменить склады → `backend/src/routes/warehouses.js`
- **«Раздел 14»** — изменить поставщиков → `backend/src/routes/suppliers.js`
- **«Раздел 15»** — изменить системные настройки → `backend/src/routes/system.js`
- **«Раздел A»** — изменить маршрутизацию/доступ к страницам → `frontend/src/App.jsx`
- **«Раздел B»** — изменить меню/навигацию → `frontend/src/components/Layout.jsx`
- **«Раздел C»** — изменить страницу пользователей → `frontend/src/pages/UsersPage.jsx`

## Добавление новой роли (инструкция)

1. **Раздел 1** (`auth.js`) — добавить роль в `ALL_ROLES` и при необходимости в `ROLE_GROUPS`
2. **Раздел A** (`App.jsx`) — добавить роль в нужные массивы `ROLES`
3. **Раздел B** (`Layout.jsx`) — добавить роль в `roleLabel` и в массивы `roles` у нужных пунктов меню
4. **Раздел C** (`UsersPage.jsx`) — добавить роль в `ROLE_CONFIG` с описанием
5. Нужные **Разделы 2–15** — добавить роль в `requireRole(...)` где нужно

## Технический стек

- **Backend**: Node.js + Express + Prisma ORM + PostgreSQL
- **Frontend**: React 18 + Vite + React Router
- **Auth**: JWT (jsonwebtoken) + bcryptjs
- **Уведомления**: Telegram Bot API (Z-отчёт при закрытии смены)
- **Деплой**: Railway / Nixpacks
